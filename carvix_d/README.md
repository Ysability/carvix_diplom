# Carvix — Система управления автопарком

<p align="center">
  <b>Дипломный проект</b> — Веб-приложение для управления расходами, ремонтами и аналитикой корпоративного автопарка
</p>

---

## Описание

**Carvix** — полнофункциональная информационная система для автоматизации процессов управления автопарком предприятия. Система охватывает полный жизненный цикл: от подачи заявки на ремонт до финансовой аналитики и прогнозирования расходов.

### Основные возможности

- **Финансовый дашборд** — KPI, динамика расходов, план/факт, топ-5 ТС по TCO
- **Аналитический дашборд** — загрузка механиков, средний срок ремонта, тренды по подразделениям
- **Прогнозирование расходов** — метод Holt-Winters (тройное экспоненциальное сглаживание) с 95% доверительным интервалом
- **Управление заявками на ремонт** — создание, назначение механика, «Автонаводка» (авто-назначение по загрузке), смена статусов
- **Управление ремонтами** — старт/закрытие ремонта, фиксация стоимости работ и запчастей
- **Реестр расходов** — объединение ремонтов, запчастей и прочих расходов; импорт из CSV
- **Бюджетирование** — плановые суммы по категориям, массовое редактирование, копирование с прошлого года
- **Приходные накладные** — учёт поступления запчастей от поставщиков
- **Управление ТС** — справочники марок/моделей, CRUD транспортных средств
- **TCO-отчёт** — совокупная стоимость владения по каждому транспортному средству
- **Экспорт** — Excel (расходы, бюджеты, TCO, аудит, аналитика) и PDF (накладные, акты списания, месячные отчёты)
- **Чат по заявкам** — обмен сообщениями между механиком и создателем заявки
- **Ролевая модель (RBAC)** — 6 ролей с гранулярным контролем доступа
- **Аудит-лог** — полная история финансовых операций

## Технологический стек

| Слой | Технологии |
|:-----|:-----------|
| **Backend** | Node.js 18+, Express 4, JWT (jsonwebtoken, bcryptjs) |
| **База данных** | PostgreSQL 14+ (пул pg, транзакции, представления) |
| **Frontend** | Vanilla JS (SPA), HTML5, CSS3 (адаптивный дизайн) |
| **Аналитика** | Holt-Winters forecast, Chart.js (на клиенте) |
| **Экспорт** | ExcelJS (xlsx), PDFKit (pdf) |
| **Тестирование** | Jest 30, Supertest |
| **Документация API** | Swagger UI (OpenAPI 3.0) |
| **Деплой** | Render (Web Service + PostgreSQL) |

│       ├── budgets.js     # Бюджеты (CRUD, bulk, copy)
│       ├── reports.js     # TCO, дашборд, аналитика, прогноз
│       ├── parts-receipts.js  # Приходные накладные
│       ├── audit.js       # Аудит-лог
│       └── exports.js     # Excel/PDF генерация
├── services/
│   └── forecast.js        # Holt-Winters прогнозирование

## Ролевая модель

| Роль | Права |
|:-----|:------|
| **Директор** | Полный доступ ко всем модулям, управление бюджетом |
| **Аналитик** | Чтение финансов, дашборды, аудит-лог, экспорт |
| **Главный механик** | Финансы (чтение/запись расходов), управление ремонтами |
| **Диспетчер** | Все заявки, назначение механиков, автонаводка |
| **Механик** | Свои ремонты (старт, закрытие), чат по заявкам |
| **Пользователь** | Создание заявок, просмотр своих ТС и заявок |

## Быстрый старт

### Предварительные требования

- **Node.js** >= 18
- **PostgreSQL** >= 14

### Установка

```bash
# 1. Клонировать репозиторий
git clone https://github.com/Ysability/carvix_diplom.git
cd carvix_diplom/carvix_d

# 2. Установить зависимости
npm install

# 3. Создать базу данных
createdb carvix

# 4. Настроить переменные окружения
cp .env.example .env
# Отредактировать .env — указать свои DB_PASSWORD и JWT_SECRET

# 5. Запустить сервер (схема и данные создадутся автоматически)
npm start
```

Приложение будет доступно по адресу: **http://localhost:3000**

### Демо-данные

При первом запуске автоматически загружаются:
- 6 ролей, 5 подразделений
- 12 сотрудников (по 2 на каждую роль)
- Транспортные средства, заявки, ремонты
- Бюджеты и прочие расходы

## Деплой на Render

Проект содержит `render.yaml` для автоматического деплоя:

```bash
# Через Render Dashboard → New → Blueprint → указать репозиторий
# Или через CLI:
render blueprint apply
```

Render автоматически:
1. Создаст PostgreSQL-базу (free tier)
2. Установит зависимости (`npm install`)
3. Запустит сервер (`node server.js`)
4. Загрузит демо-данные при первом старте
